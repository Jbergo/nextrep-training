# NextRep Training

# Características del proyecto

## Nombre

NextRep Training

## Definición del proyecto

El proyecto consiste en el desarrollo de una aplicación web para una empresa ficticia llamada NextRep Training, dedicada a la venta de servicios de entrenamiento online. La plataforma estará orientada a usuarios que desean mejorar su condición física mediante programaciones de entrenamiento adaptadas a distintos niveles, objetivos y perfiles personales.

La finalidad de la aplicación será ofrecer un entorno digital en el que los usuarios puedan consultar los diferentes planes disponibles, conocer sus características principales y solicitar la contratación del servicio que mejor se ajuste a sus necesidades. Al mismo tiempo, el sistema permitirá gestionar internamente la información de los servicios y las solicitudes realizadas por los clientes.

Esta propuesta encaja con el planteamiento general del proyecto intermodular, ya que se trata de un **portal de servicios** realista, coherente y viable para 1.º de DAW.

## Tipo de empresa

NextRep Training representa una **empresa de servicios deportivos online** especializada en la creación y comercialización de programaciones de entrenamiento para distintos tipos de usuarios.

No se trata de una tienda de productos físicos, sino de un negocio digital centrado en la prestación de servicios personalizados o semipersonalizados relacionados con el entrenamiento y la planificación deportiva.

## Para qué sirve

- Presentar la empresa y su propuesta de valor.
- Mostrar un catálogo de servicios de entrenamiento online.
- Explicar cada servicio de forma clara.
- Permitir que un cliente solicite la contratación de un plan.
- Registrar dichas solicitudes en una base de datos.
- Facilitar una gestión básica de servicios y contrataciones desde una parte administrativa.

## A quién va dirigido

La web está dirigida a personas que desean seguir un plan de entrenamiento online sin necesidad de acudir presencialmente a un centro deportivo, especialmente:

- Usuarios principiantes que necesitan una guía inicial.
- Personas con objetivo de pérdida de grasa o mejora de la condición física.
- Usuarios intermedios que buscan planes de fuerza o hipertrofia.
- Personas con poco tiempo que prefieren entrenar con planificación online.
- Clientes que quieren un servicio estructurado, accesible y adaptable a su nivel.

### Público objetivo resumido

Adultos jóvenes y de mediana edad interesados en mejorar su condición física mediante programas de entrenamiento online personalizados.

## Problema que resuelve

Actualmente muchas personas quieren entrenar, pero se encuentran con varios problemas:

- Falta de conocimientos para organizar una rutina adecuada.
- Exceso de información desordenada o poco fiable en internet.
- Dificultad para encontrar un plan adaptado a su nivel real.
- Falta de tiempo para acudir a un centro deportivo.
- Coste elevado de ciertos servicios presenciales.

NextRep Training resuelve este problema ofreciendo un sistema sencillo para consultar y contratar programaciones de entrenamiento online organizadas por objetivos y niveles.

## Servicios que ofrece

La empresa ofrece servicios de programación de entrenamiento online.

Estos servicios consisten en planes diseñados según:

- Nivel del usuario.
- Objetivo principal.
- Duración del programa.
- Tipo de entrenamiento.
- Grado de personalización.

## Propuesta de valor

Ofrecer una solución digital clara y accesible para contratar planes de entrenamiento online, evitando procesos complejos y facilitando al usuario la elección de un servicio adecuado a su perfil. De esta forma, permite transformar una necesidad difusa —“quiero entrenar pero no sé cómo organizarme”— en un servicio concreto, entendible y gestionable.

## Objetivos

### General

Desarrollar una aplicación web funcional para una empresa ficticia de entrenamiento online que permita mostrar sus servicios, registrar solicitudes de contratación y gestionar esta información mediante una base de datos relacional y una aplicación Java conectada con JDBC.

### Específicos

- Diseñar una web corporativa orientada a la venta de servicios.
- Mostrar un catálogo de planes de entrenamiento online.
- Permitir consultar el detalle de cada servicio.
- Implementar un formulario de contratación o reserva.
- Registrar clientes y contrataciones en una base de datos relacional.
- Desarrollar una parte de administración para gestionar servicios y pedidos.
- Aplicar una estructura limpia y coherente entre frontend, backend y base de datos.
- Documentar el proyecto de forma profesional para su presentación académica.

## Tecnologías

- **HTML5**, para la estructura y el contenido de las páginas web.
- **CSS3**, para el diseño visual, la maquetación y la adaptación responsive.
- **JavaScript**, para aportar interactividad básica en formularios y componentes de la interfaz.
- **Java**, como lenguaje principal para la lógica de la aplicación.
- **JDBC**, para la conexión entre la aplicación Java y la base de datos.
- **SQL**, para la creación, consulta y gestión de los datos.
- **MySQL** o **MariaDB**, como sistema gestor de base de datos relacional.
- **Git y GitHub**, para el control de versiones y la organización del repositorio del proyecto.

## Justificación de viabilidad

Se trata de un proyecto viable para 1.º de DAW porque permite integrar de forma coherente contenidos de varios módulos: diseño web, bases de datos, programación, documentación y control de versiones. Además, el alcance puede ajustarse fácilmente a un MVP funcional sin necesidad de integrar sistemas externos complejos ni pagos reales.

La idea también resulta adecuada para la fase posterior de diseño de base de datos, ya que el negocio permite identificar entidades reales como clientes, servicios, contrataciones y usuarios administradores.
